2024 · IoT · Embedded Systems

Gas Level Monitoring System

A real-time embedded safety system that continuously monitors ambient gas concentration and triggers audible and visual alerts the moment unsafe levels are detected — preventing accidents before they happen.

Arduino UNOArduino IDEMQ Gas SensorBuzzerLCD 16x2Embedded C

Project Overview

The Gas Level Monitoring System is a hardware-software integrated safety solution designed to detect dangerous gas concentrations in enclosed spaces like kitchens, labs, and industrial environments. Using an MQ series gas sensor connected to an Arduino UNO, the system continuously reads analog gas concentration values and compares them against a safe threshold.

How It Works

  • The MQ gas sensor detects LPG, methane, and other combustible gases and outputs an analog voltage proportional to concentration.
  • The Arduino UNO reads this value via its ADC and converts it to a PPM (parts per million) reading.
  • The LCD display shows the real-time gas level and status ("SAFE" / "DANGER") at all times.
  • When gas concentration exceeds the defined threshold (e.g. 300 PPM), the buzzer is activated and the LCD flashes a warning.
  • The alert continues until gas levels return to safe range, ensuring sustained awareness.

Key Features

  • Real-time continuous monitoring with instant response (sub-second latency)
  • Visual display on LCD 16×2 showing live readings
  • Audible buzzer alert for immediate physical warning
  • Adjustable threshold via potentiometer for different environments
  • Low-power, low-cost design suitable for home or industrial use

Challenges & Learnings

Calibrating the gas sensor for accurate PPM readings required understanding its voltage-to-concentration curve from the datasheet. Noise filtering in the ADC readings was handled by averaging multiple samples per cycle. This project deepened my understanding of analog sensor interfacing, embedded C programming, and real-world safety system design.

2025 · IoT · Smart City

Smart Sewage Monitoring System

An IoT-based infrastructure monitoring solution that detects toxic gas buildup and sewage blockages in underground drain systems, sending real-time SMS alerts to municipal workers — making urban sanitation safer and smarter.

Arduino UNOGas SensorUltrasonic SensorGSM SIM800LLCD 16x2IoT

Project Overview

Sewage workers face extreme risks from toxic gases like hydrogen sulfide (H₂S) and methane that accumulate in manholes and underground drains. This project provides a low-cost, automated monitoring solution that eliminates the need for workers to manually check conditions before entering — protecting lives through technology.

How It Works

  • An MQ gas sensor continuously monitors for toxic and combustible gases inside the sewage chamber.
  • An ultrasonic sensor measures the sewage water level to detect blockages or overflow conditions.
  • Both readings are processed by the Arduino UNO in real time.
  • The LCD displays live gas levels and water depth readings on site.
  • When either reading exceeds safe thresholds, the GSM SIM800L module sends an SMS alert with the specific hazard type to registered mobile numbers.
  • The system remains active 24/7 on a low-power loop, transmitting alerts only when needed.

Key Features

  • Dual-sensor monitoring — gas toxicity + sewage depth simultaneously
  • Real-time GSM SMS notifications to multiple registered numbers
  • Onsite LCD status display for maintenance personnel
  • 24/7 autonomous operation — no human intervention needed
  • Scalable design — multiple units can be deployed across a city grid

Impact & Learnings

This project directly addresses a critical urban safety problem. Integrating GSM communication with Arduino taught me about UART serial communication, AT command protocols, and the challenges of reliable communication in low-signal underground environments.

E-Commerce Web Application

A fully functional full-stack shopping platform featuring a responsive React.js frontend, Node.js backend API, and database integration — delivering a seamless product browsing, cart, and checkout experience.

React.jsNode.jsJavaScriptSQL DatabaseREST APIHTML/CSS

Project Overview

Built during my internship at Krishtec, this e-commerce web application demonstrates a complete full-stack implementation. The platform allows users to browse a product catalogue, add items to a cart, and complete purchases — with all data persisted in a relational database. It was built from scratch over 15 days of intensive development.

Architecture

  • Frontend (React.js): Component-based UI with React Router for client-side navigation, useState/useEffect hooks for state management, and responsive CSS layout for mobile/desktop.
  • Backend (Node.js): Express.js REST API handling product listing, cart operations, and order processing. Routes structured with MVC pattern.
  • Database (SQL): Relational schema with tables for products, users, orders and order items. Queries optimised for listing and cart operations.
  • API Integration: Axios used on the frontend for HTTP calls to the backend REST endpoints.

Key Features

  • Product catalogue with category filtering and search
  • Add to cart, update quantity, remove items
  • Order placement with total price calculation
  • Responsive layout working on mobile and desktop
  • Backend validation and error handling on all API endpoints

Learnings

This project taught me how React component architecture scales to real applications, how to design a REST API with Express, and how frontend and backend connect through HTTP. Working under real internship conditions with deadlines gave me practical experience in time management and code organisation.

2025 · Computer Vision · Machine Learning

Driver Drowsiness Detection System

A real-time AI-powered safety system that monitors a driver's eye and facial cues through a camera, detects drowsiness using computer vision and machine learning, and triggers instant alerts to prevent road accidents.

PythonOpenCVMachine LearningdlibCamera ModulePygame (Alert)

Project Overview

Driver fatigue is one of the leading causes of road accidents worldwide. This system uses a standard webcam and computer vision algorithms to continuously analyse the driver's face and detect early signs of drowsiness — specifically by monitoring eye aspect ratio (EAR) — and sounds an alert before the driver falls asleep at the wheel.

How It Works

  • Face Detection: OpenCV's Haar Cascade or dlib's HOG-based face detector locates the driver's face in each video frame.
  • Facial Landmark Detection: dlib's 68-point landmark predictor maps key facial points, isolating the eye regions precisely.
  • Eye Aspect Ratio (EAR): The EAR formula calculates the ratio of eye height to width. When eyes are open the EAR is ~0.3; when closed it drops sharply towards 0.
  • Drowsiness Threshold: If EAR stays below 0.25 for more than 20 consecutive frames (~0.6 seconds), drowsiness is flagged.
  • Alert System: A loud audio alert is triggered via Pygame, and a visual warning overlays the video feed.

Key Features

  • Real-time video processing at 20+ FPS on standard hardware
  • Works in varied lighting conditions with preprocessing
  • Immediate audio + visual alert on drowsiness detection
  • No specialised hardware required — uses any standard webcam
  • Configurable sensitivity threshold for different driver profiles

Challenges & Learnings

The biggest challenge was handling false positives — detecting blinks as drowsiness. This was solved by requiring the low-EAR condition to persist for multiple consecutive frames rather than a single frame. Working with dlib's pretrained models and understanding facial geometry were major learning milestones.
